# Цифрова експозиція музею

Доброго часу доби!  
Це простий гайд для людей, які **не програмують**.  
Ви зможете самостійно наповнювати цифрову колекцію та оновлювати сайт.

---

## 1. Перші кроки на GitHub

1. Увійдіть у свій акаунт на [github.com](https://github.com)
2. Натисніть кнопку **«New repository»**
3. Заповніть:
   - **Repository name**: `museum-collection`
   - **Description**: «Цифрова експозиція [назва вашого музею]»
   - Оберіть **Public**
   - Поставте галочку **«Add a README file»**
4. Натисніть **Create repository**

---

## 2. Структура папок (створіть один раз)

Після створення репозиторію структура має бути такою:

museum-collection/
├── index.html
├── data.json
├── LICENSE
├── README.md
└── assets/
├── photos/          ← звичайні фото експонатів (.jpg) - 2D
├── normalmaps/      ← карти нормалей, (.png) - 2.5D
├── models/          ← 3D-моделі (.glb) - 3D
└── animations/      ← GIF-анімації (.gif) - 4D

---

## 3. Правила називання файлів (обов’язково!)

| Тип файлу                  | Папка              | Формат назви                              | Приклад файлу                     |
|----------------------------|--------------------|-------------------------------------------|-----------------------------------|
| **Звичайне фото**          | `photos/`          | `Інвентарний_номер_01.jpg`               | `МК-001_01.jpg`                   |
| **Додаткове фото**         | `photos/`          | `Інвентарний_номер_02.jpg`               | `МК-001_02.jpg`                   |
| **Карта нормалей**         | `normalmaps/`      | `Інвентарний_номер_normal.png`           | `МК-001_normal.png`               |
| **3D-модель**              | `models/`          | `Інвентарний_номер.glb`                  | `МК-001.glb`                      |
| **Анімація / відео**       | `animations/`      | `Інвентарний_номер.gif`                  | `МК-001.gif`                      |

**Важливі правила:**
- Інвентарний номер **завжди на початку** назви файлу.
- Використовуйте тільки латинські літери, цифри, `-` та `_`.
- **Заборонено**: пробіли, українські літери в назвах файлів, кирилиця.

---

## 4. Заповнення Excel-таблиці

1. Завантажте шаблон `Museum-Collection-Template.xlsx`
2. Заповнюйте по одному експонату в один рядок.

**Ключові колонки:**
- `category` — обов’язково з номером: `01-херсонес`, `02-скіфи`, `03-салтівська`…
- `sortOrder` — число з кроком 10 (1010, 1020, 1030…)
- `2D` і `2.5D` — кілька файлів через `;` (крапка з комою)

**Рекомендація:**  
В одній категорії бажано **не більше 900 експонатів**. Якщо перевищуєте — створіть нову категорію з наступним номером.

---

## 5. Як згенерувати data.json

1. Заповніть таблицю.
2. Натисніть кнопку **«Згенерувати data.json»** на аркуші.
3. Готовий JSON автоматично скопіюється в буфер обміну.
4. Відкрийте `data.json` на GitHub → Edit → вставте (Ctrl+V) → Commit changes.

---

## 6. Завантаження файлів

1. Перейдіть у потрібну папку (`photos`, `normalmaps`, `models` тощо).
2. Натисніть **Add file → Upload files**.
3. Перетягніть файли.
4. Внизу напишіть коментар (опційно) і натисніть **Commit changes**.

---

## 7. Налаштування index.html (один раз)

Відкрийте файл `index.html` і замініть:

- `Назва музею` → повна назва вашого музею
- `Колекція` → якщо хочете іншу назву експозиції (можна лишити "Колекція")
- `Назва колекції` → короткий підпис під заголовком (або видаліть цей рядок)
- Посилання кнопки **«Головна»** → на головну сторінку вашого сайту

---

## 8. Як додавати нові експонати

1. Відкрийте Excel-таблицю.
2. Додайте новий рядок внизу.
3. Заповніть дані.
4. Натисніть кнопку «Згенерувати data.json».
5. Оновіть файл `data.json` на GitHub.
6. Завантажте нові файли (картинки, карти нормалей, моделі .glb, gif-анімації) в папки `assets/…`
***Дані полів: 2.5D, 3D, 3D-II, 4D, 🔗 - не є обв'язковими до заповнення

Сайт оновиться автоматично протягом кількох секунд.
